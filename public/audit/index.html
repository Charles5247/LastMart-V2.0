The Investor-Ready Narrative

LastMart is a hyper-local marketplace built on the insight that Nigeria's ₦42 trillion informal commerce market — dominated by WhatsApp vendors, Instagram sellers, and unorganized local markets — is ready to be structured into a fast, trusted digital experience. 🚨 Critical Issues (Fix Before Any Demo)

Critical
Cart Is Hidden in Dashboard
The shopping cart is only accessible via /dashboard/customer — not a dedicated /cart page. This is counter-intuitive to every e-commerce norm. Users who add to cart have no obvious place to go.
💥 Impact: Estimated 30–40% cart abandonment increase. Fix: Create /cart page accessible from navbar cart icon.
Critical
Homepage is 'use client' — Zero SEO
The homepage (page.tsx) has 'use client' directive. This disables React server-side rendering for the most important page. Google cannot index the content that matters.
💥 Impact: Zero organic traffic. Fix: Separate data fetching (Server Component) from interactive parts (Client Components).
Critical
No Page-Specific Meta Tags / Open Graph
Every page shares the same generic metadata from layout.tsx. Product pages, vendor pages, city pages — all have identical titles and descriptions.
💥 Impact: 0% SEO benefit from product/vendor pages. Fix: Add generateMetadata() to each page. Takes 1 day.
Critical
No sitemap.xml or robots.txt
There is no /public/robots.txt and no /api/sitemap.xml route. Google's crawler has no guidance on what to index.
💥 Impact: Pages may not be indexed at all. Fix: Generate dynamic sitemap from product/vendor routes. Takes 4 hours.
Critical
Auth Token in localStorage (Security Risk)
JWT tokens are stored in localStorage — vulnerable to XSS attacks. For an e-commerce platform handling payments, this is a significant security risk that any technical investor will flag.
💥 Impact: Security red flag for investors. Fix: Migrate to httpOnly cookies via /api/auth/session endpoint.
Critical
No Guest Checkout Option
Checkout requires a logged-in account. Forcing registration before purchase is a documented conversion killer — can reduce conversions by 25–35% vs. guest checkout.
💥 Impact: Massive checkout drop-off. Fix: Allow phone + email guest checkout with account creation post-order.

🚩 Investor Red Flags
Issues That Will Kill
Your Funding Conversation

These are the specific things a technical investor or accelerator scout will immediately notice. Fix these before any pitch.

🚩 No Traction Metrics Visible CRITICAL
There are no live user counts, vendor counts, GMV, or orders processed visible anywhere — not on the homepage, not in any public dashboard. Investors need to see proof of traction, even at an early stage. The admin analytics exist but aren't surfaced.
✅ Fix: Add a live "social proof bar" on homepage: "X vendors · Y products · Z cities served · ₦X GMV processed"
🚩 JWT in localStorage (XSS Vulnerability) CRITICAL
Any security-minded investor or technical advisor will immediately flag that auth tokens are stored in localStorage, not httpOnly cookies. This is a known XSS vulnerability — if any third-party script is compromised, all user sessions can be hijacked.
✅ Fix: Migrate to httpOnly cookie sessions. ~1 day of work. Non-negotiable for a payment-processing platform.
🚩 "Nigeria's #1" Claim With No Data MAJOR
The hero badge says "Nigeria's #1 Local Market" and the second slide claims "1000+ Verified Vendors". If these aren't real numbers, this destroys credibility in due diligence. Investors check claims.
✅ Fix: Only use real, verifiable numbers. Change to honest messaging: "Connecting local vendors to customers faster" until you have real numbers to show.
🚩 All Support Links Are Dead (#) MAJOR
Help Center, Track My Order, Returns Policy, Payment Options, Contact Support, Terms of Service, Privacy Policy, Cookie Policy, Seller Agreement — all are placeholder links. This is a legal liability and a massive trust issue, especially for a platform handling Nigerian payments.
✅ Fix: Create actual pages or at minimum link to a WhatsApp support number and a proper Terms/Privacy page.
🚩 No Revenue Model Transparency MAJOR
The platform has a ranking/ads system and referral commissions built, but nowhere does it explicitly state how LastMart makes money per transaction. There's no commission percentage shown to vendors. Investors will ask "what's your take rate?"
✅ Fix: Add a revenue model section to the /sell page: "We take X% per completed sale" + "Premium plans start at ₦X/month"
🚩 Demo Accounts Publicly Visible on Login MINOR
The login page has "Quick Demo Login" buttons with hardcoded credentials (admin@lastmart.com / admin123, vendor, customer). While useful for demos, this signals a lack of production discipline. Remove before live demo with investors.
✅ Fix: Move demo accounts to a separate demo environment or hide behind a flag. Add note "Demo environment" if keeping.
🚩 Social Media Links Are Dead MINOR
Facebook, Twitter, Instagram, and YouTube icons in the footer link to "#". An investor who checks social presence (standard in due diligence) will see no community.
✅ Fix: Create and link real LastMart social accounts immediately. Even empty accounts are better than dead links.
🚩 SQLite in Production Risk MINOR
The README indicates SQLite as the development database but production-readiness requires PostgreSQL. If the demo environment is running on SQLite, it will not survive concurrent users during a live demo.
✅ Fix: Ensure demo runs on PostgreSQL. Document the migration path clearly in your pitch deck tech section.

    Key UI/UX Principles for LastMart

    📱 Mobile-First Priority Critical

    Nigeria has 90%+ mobile internet usage. Every new component must be designed for 375px first. Use the existing .grid-products and .scroll-row utilities, but audit every page on iPhone SE size.

    ⚡ Performance Budget Critical

    Nigerian internet speeds average 20–30Mbps mobile. Target: First Contentful Paint under 1.5s. Use Next.js Image component for all product images. Lazy-load below-fold content. Compress all OG images.

    🎨 Design Consistency Enhancement

    The globals.css design tokens are well-structured (--brand, --text-primary etc). Enforce them more consistently — some pages use hardcoded Tailwind colors instead of CSS vars. Create a component library doc.

    🌍 Localization UX Strength

    EN/YO/HA/IG support is a genuine differentiator. The LanguageSwitcher component exists — make it more prominent. Add it to the mobile menu and checkout flow. This is a key story for pan-Nigeria expansion.

    ✨ Micro-interactions Polish

    ProductCard hover effect (image swap + add-to-cart overlay) is excellent. Extend this pattern: VendorCard hover should show "View X products" overlay. Cart icon should animate on add. Checkout steps should have progress animation.

    ♿ Accessibility Important

    The :focus-visible ring is defined in CSS (good). Ensure all interactive elements have aria-labels, especially icon-only buttons. Add alt text to all product images. Form inputs need associated labels (currently using placeholder only in some places).
